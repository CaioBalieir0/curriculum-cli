# Original Resume Layout Design

## Goal

Make PDFs generated by the CLI match `curriculo-caio-balieiro.pdf` visually and structurally, starting with the Portuguese output generated from `data/default-pt.json` and `templates/pt.html`.

The generated Portuguese PDF should no longer use the current generic resume layout. It should reproduce the original resume's hierarchy, section names, colors, spacing, typography, and content grouping as closely as practical with the existing HTML/CSS + Puppeteer renderer.

## Current State

The CLI renders language-specific Handlebars templates through Puppeteer in `src/render.ts`.

The current `templates/pt.html` differs from the original PDF in several important ways:

- Header is left-aligned instead of centered.
- Name uses title case and dark color instead of uppercase, centered text.
- Section headings are black with gray rules instead of blue headings and blue dividers.
- Skills are rendered as a two-column grid instead of compact inline rows.
- Education and languages are separate sections instead of one combined final section.
- Extracted text from the generated PDF shows content grouping issues where bullets can appear away from their related entries.

## Selected Approach

Recreate the original layout in the existing HTML templates and keep the current Puppeteer rendering pipeline.

This is the smallest correct change because the project already uses HTML/CSS templates, which are sufficient for matching the original Google Docs-style PDF. It avoids adding a theme system or replacing the rendering engine.

## Layout Requirements

The Portuguese generated PDF should match the original structure:

- Centered header.
- Uppercase name: `CAIO BALIEIRO MARIANO`.
- Blue professional title below the name.
- Contact details centered with `•` separators and links on a second line.
- Section labels matching the original:
  - `RESUMO PROFISSIONAL`
  - `HABILIDADES TÉCNICAS`
  - `EXPERIÊNCIA PROFISSIONAL`
  - `PROJETOS PESSOAIS`
  - `FORMAÇÃO & IDIOMAS`
- Blue headings and blue horizontal rules.
- Compact body text with page density close to the original.
- Skills rendered as compact inline rows, not as a card/grid layout.
- Experience entries rendered with title, role, type/date, context, and bullets grouped together.
- Projects rendered before the final education/languages block.
- Education, certifications, and languages rendered together in the final section.

## Data Shape Considerations

The existing data shape can mostly support the target layout:

- `profile` supplies name, title, summary, location, phone, email, LinkedIn, and GitHub.
- `skills` supplies category and item lists.
- `experience` supplies title, context, and bullets.
- `projects` supplies title, context, and bullets.
- `education.details` currently contains certifications.
- `languages` contains language entries.

The template may parse title strings visually by rendering the stored title as-is, but no schema change is required for this layout pass. If exact title styling requires splitting organization, role, contract type, and dates, that can be done later with a data model change, but it is not required for the initial reproduction.

## Renderer Requirements

Keep `src/render.ts` using Puppeteer with A4 output and `printBackground: true`.

Margins may be adjusted from the current `12mm` if needed to match the original PDF's content width and vertical placement. Any renderer change should be limited to print options that affect visual fidelity.

## English Template

Apply the same visual system to `templates/en.html` so generated PDFs have a consistent style across languages. English section labels should remain translated, but the layout should follow the original Portuguese resume style.

## Verification

Verification should use the current CLI flow:

- Build the TypeScript project.
- Generate the Portuguese PDF with `node ./dist/cli.js generate-pt --output caio-pt.pdf`.
- Convert `curriculo-caio-balieiro.pdf` and `output/caio-pt.pdf` with `pdftoppm -png -r 150`.
- Compare the resulting page images visually for header alignment, section styling, skills layout, content grouping, pagination, and final education/languages section.

The target is a close practical match to the original PDF, not byte-for-byte PDF equivalence.

## Non-Goals

- Do not replace Puppeteer.
- Do not add a theme flag or multiple selectable layouts.
- Do not change the CLI command names or user-facing flags.
- Do not introduce a new PDF generation library.
- Do not require screenshots from the user unless `pdftoppm` comparison proves insufficient.
